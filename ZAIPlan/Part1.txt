# PR Advisory Agent Blueprint: Letta Code + Kubernetes

## Executive Summary
This document outlines the architecture and implementation plan for an AI-driven PR Advisory Agent. The agent comments on every Pull Request, analyzing dependency versions and suggesting architectural alternatives. 

The design leverages **Letta Code** for its built-in agent harness and GitHub Action integration, ensuring strict advisory-only boundaries. State and memory are persisted using a **Self-Hosted Letta Server on Kubernetes**, utilizing Letta's **MemFS** (git-backed filesystem) for cross-PR learning, and **Sleep-Time Dream Agents** for automated self-evaluation and memory consolidation.

---

## 1. Requirements Mapping

| Requirement | Implementation in this Design |
| :--- | :--- |
| **PR Coverage** | GitHub Action triggers on every `pull_request: [opened]` event. |
| **Advisory Only** | Letta Code GitHub Action natively blocks PR approvals and workflow file modifications. Agent lacks write access to source code. |
| **Dependency Analysis** | Custom `check_deps.py` script executed via Letta's shell tool. |
| **Version Checking** | Custom `augmented_web_search` (Exa + Tavily + Firecrawl) and Context7 MCP for library docs. |
| **Architectural Alternatives** | LLM reasoning guided by the `architecture-advisory` skill to generate 5 alternatives with pros/cons. |
| **Agent-Driven** | Letta Code provides a full agent harness with LLM reasoning, not a rigid script. |
| **Fully-Featured Harness** | Letta Code natively provides file read, shell execution, and PR commenting tools. |
| **Persistent Memory** | MemFS (git-backed filesystem) stored on a Kubernetes Persistent Volume (PVC). |
| **Periodic Self-Evaluation** | Letta's built-in Sleep-Time Dream Agents (compaction-triggered) reflect on recent PRs and consolidate memory. |
| **Rapid Implementation** | 1-day, 4-stage deployment plan leveraging existing K8s infrastructure and GitHub Actions. |

---

## 2. Architecture Overview

This design is fully self-hosted and event-driven. By routing GitHub webhooks directly to the self-hosted Letta Server on your Kubernetes cluster, we eliminate the need for a GitHub Actions runner entirely. The agent runs inside Kubernetes, clones the repository dynamically, runs the audit tools locally, posts the advisory comment, and cleans up.

```mermaid
flowchart TD
    A[GitHub Pull Request Event] -->|Webhook Event| B[Kubernetes Ingress / TLS]
    B --> C[Webhook Handler Sidecar / Letta API]
    
    C --> D[Letta Server on Kubernetes]
    D --> E[Persistent Volume<br/>MemFS Git Repository]
    
    D --> F[Agent Instance]
    F --> G[Tools: git_clone, check_deps, OSV, Web Search]
    F --> H[Skills: audit, advisory]
    F --> I[Sleep-Time Dream Agent<br/>Background consolidation]
    
    F -->|Clones PR Branch| J[Local Pod Workspace]
    F -->|Runs check_deps/OSV| J
    
    F --> K[Posts Advisory Comment<br/>via GitHub API]
    K --> L[Updates MemFS on PV]
```

### Core Components
1. **GitHub Webhook:** Configured at the repository or organization level to forward `pull_request` events directly to your Kubernetes ingress endpoint.
2. **Webhook Listener / API Gateway:** Receives the webhook, authenticates the GitHub signature, and translates it into a message query for the Letta Agent.
3. **Letta Server (K8s Deployment):** The agent runtime. Hosted on your Kubernetes cluster to maintain persistent state.
4. **MemFS (Memory File System):** Letta's git-backed memory substrate stored on a K8s Persistent Volume (PVC).
5. **Sleep-Time Dream Agent:** A built-in background process that wakes up on context compaction triggers to evaluate recent PRs and consolidate memory patterns.
6. **Custom Git Clone Tool:** A tool given to the agent to clone the target PR repository branch locally into the pod's temporary space.

## 3. Implementation Plan (1-Day, 4 Stages)

### Stage 1: Kubernetes Deployment & Storage (Hours 1-4)
**Goal:** Deploy the Letta Server to your cluster with persistent storage.

1. **Create Kubernetes Manifests:**
   * **Deployment:** Run the `letta/letta-server` Docker image along with a webhook handler sidecar.
   * **Service:** Expose port 8283 and webhook receiver port internally.
   * **PersistentVolumeClaim (PVC):** Provision storage for the MemFS git repository.
2. **Configure Ingress:** Set up an Ingress route with TLS to expose the webhook receiver securely (e.g., `https://letta-webhooks.yourdomain.com/github`).
3. **Initialize MemFS:** Exec into the pod and run `/memfs enable` to initialize the git-backed memory on the persistent volume.
4. **Quality Gate 1:** Verify the pod is running, the PVC is bound, and you can reach the server endpoints externally.

### Stage 2: Webhook & Secret Provisioning (Hours 5-8)
**Goal:** Configure GitHub to trigger the self-hosted agent directly.

1. **Create K8s Secrets:** Store your Letta API key, Webhook Verification Secret, and GitHub PAT/App Installation Token (`GITHUB_TOKEN`) in Kubernetes.
2. **Configure GitHub Webhook:** Register a webhook in your GitHub repository pointing to your Ingress URL. Select event: `Pull requests`.
3. **Define Agent & Skills:** Via the Letta CLI connected to your server, create the agent and attach the `dependency-audit` and `architecture-advisory` skills.
4. **Quality Gate 2:** Trigger a test pull request. Verify that the webhook payload is delivered to Kubernetes, is parsed correctly, and triggers the agent on your server.

### Stage 3: Tooling & Dynamic Workspace Cloning (Hours 9-12)
**Goal:** Implement the repository cloning tool and custom audit scripts.

1. **Containerize Tools:** Create a custom Dockerfile that extends the Letta server image and includes `git`, `python3`, `pip`, `osv-scanner`, and your `check_deps.py` script.
2. **Implement Clone Tool:** Write the `git_clone` tool logic that enables the agent to dynamically clone the incoming PR's branch to `/tmp/workspace` using the injected GitHub token.
3. **Register Tools:** Register the `git_clone` tool, `check_deps.py`, `osv-scanner`, and search tools (`augmented_web_search`) to the agent.
4. **Quality Gate 3:** Trigger a PR with an outdated dependency. Verify the agent clones the branch, runs the scripts locally, posts the comment, and cleans up the cloned files.

### Stage 4: Memory Consolidation & Polish (Hours 13-16)
**Goal:** Enable sleep-time consolidation and verify persistent memory.

1. **Configure Sleep-Time:** Run `/sleeptime compaction` in the Letta CLI to enable the dream agent.
2. **Test Memory Persistence:** Open PR #1. Wait for sleep-time consolidation. Open PR #2 referencing the same dependency. Verify the agent recalls the context from PR #1.
3. **Monitor K8s Logs:** Check the pod logs to ensure the sleep-time agent is running without errors and committing to the MemFS git repo on the persistent volume.
4. **Final Polish:** Refine the system prompt to ensure the 5 architectural alternatives are strictly advisory and well-formatted.

### Appendix A: Webhook Receiver Listener (`webhook-handler.py`)
*This runs as a lightweight service or sidecar in the Kubernetes pod to receive GitHub webhooks and trigger the Letta Agent via its REST API.*

```python
from flask import Flask, request, jsonify
import hmac
import hashlib
import os
import requests

app = Flask(__name__)

LETTA_API_URL = "http://localhost:8283/v1/agents"
AGENT_ID = os.environ.get("LETTA_AGENT_ID")
LETTA_API_KEY = os.environ.get("LETTA_API_KEY")
WEBHOOK_SECRET = os.environ.get("GITHUB_WEBHOOK_SECRET").encode()

@app.route("/github", methods=["POST"])
def github_webhook():
    # Verify GitHub Webhook Signature
    signature = request.headers.get("X-Hub-Signature-256")
    if not signature:
        return "Missing signature", 401
    
    sha_name, signature = signature.split("=")
    mac = hmac.new(WEBHOOK_SECRET, request.data, hashlib.sha256)
    if not hmac.compare_digest(mac.hexdigest(), signature):
        return "Invalid signature", 401

    payload = request.json
    action = payload.get("action")
    
    # We trigger the agent when a PR is opened or synchronized (updated)
    if "pull_request" in payload and action in ["opened", "synchronize"]:
        pr = payload["pull_request"]
        repo = payload["repository"]
        
        # Build prompt payload for the agent
        agent_payload = {
            "message": f"Review PR #{pr['number']} in {repo['full_name']}. Branch: {pr['head']['ref']}. Clone URL: {pr['head']['repo']['clone_url']}"
        }
        
        # Trigger the agent turn via Letta API
        headers = {"Authorization": f"Bearer {LETTA_API_KEY}"}
        r = requests.post(f"{LETTA_API_URL}/{AGENT_ID}/messages", json=agent_payload, headers=headers)
        return jsonify({"status": "triggered", "agent_response": r.status_code})

    return "Event ignored", 200

if __name__ == "__main__":
    app.run(host="0.0.0.0", port=5000)
```

### Appendix B: `check_deps.py` (Custom Dependency Tool)
```python
#!/usr/bin/env python3
"""Scan repo for dependencies, check latest versions."""
import json
import sys
import requests
from pathlib import Path

def check_npm(repo_path):
    pkg_json = Path(repo_path) / "package.json"
    if not pkg_json.exists(): return []
    deps = json.load(open(pkg_json)).get("dependencies", {})
    results = []
    for name, current_ver in deps.items():
        try:
            r = requests.get(f"https://registry.npmjs.org/{name}/latest", timeout=5)
            latest_ver = r.json().get("version")
            if latest_ver and latest_ver != current_ver:
                results.append({
                    "ecosystem": "npm",
                    "name": name,
                    "current": current_ver,
                    "latest": latest_ver,
                    "update_available": True
                })
        except Exception:
            continue
    return results

# Add check_pypi, check_go_mod, etc. as needed

if __name__ == "__main__":
    repo = sys.argv[1]
    all_deps = check_npm(repo) # + check_pypi(repo)
    print(json.dumps(all_deps, indent=2))
```
