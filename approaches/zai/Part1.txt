# PR Advisory Agent Blueprint: Letta Code + Kubernetes

## Executive Summary
This document outlines the architecture and implementation plan for an AI-driven PR Advisory Agent. The agent comments on every Pull Request, analyzing dependency versions and suggesting architectural alternatives.

The design leverages **Letta Code** for its built-in agent harness, ensuring strict advisory-only boundaries. State and memory are persisted using a **Self-Hosted Letta Server on Kubernetes**, utilizing Letta's **MemFS** (git-backed filesystem) alongside **memory blocks** (always-in-context directives) for cross-PR learning, and **Sleep-Time Dream Agents** for automated self-evaluation and memory consolidation.

> **Deprecation Notice:** The `letta/letta` Docker image is no longer actively maintained or supported as a Letta product surface (per official docs: https://docs.letta.com/guides/docker/). This plan uses it because self-hosting on K8s requires it, but you should accept the maintenance risk. For alternatives, consider Letta Code local mode (`letta --backend local`) on a persistent machine accessible via Tailscale.

---

## 1. Requirements Mapping

| Requirement | Implementation in this Design |
| :--- | :--- |
| **PR Coverage** | GitHub webhook forwards `pull_request: [opened, synchronize]` events directly to the K8s ingress. |
| **Advisory Only** | Agent's system prompt enforces advisory-only behavior. GitHub token scoped to `pull-requests: write` only (no `contents: write`). Agent cannot push code or approve PRs. |
| **Dependency Analysis** | Custom `check_deps.py` script covering npm, pip, cargo, go, and ruby. Executed via Letta's shell tool inside the K8s pod. |
| **Version Checking** | Direct package registry API queries (npm, PyPI, crates.io, etc.) plus optional `augmented_web_search` (Exa + Tavily) for changelog context. |
| **Architectural Alternatives** | LLM reasoning guided by the `architecture-advisory` skill to generate 5 alternatives with pros/cons. |
| **Agent-Driven** | Letta Code provides a full agent harness with LLM reasoning, not a rigid script. |
| **Fully-Featured Harness** | Letta Code natively provides file read, shell execution, and PR commenting tools. |
| **Persistent Memory** | MemFS (git-backed filesystem) stored on a Kubernetes Persistent Volume (PVC), coexisting with memory blocks (always-in-context agent directives). |
| **Periodic Self-Evaluation** | Letta's built-in Sleep-Time Dream Agents (compaction-triggered) reflect on recent PRs and consolidate memory. |
| **Rapid Implementation** | 1-day, 5-stage deployment plan (including Stage 0: Agent Initialization) leveraging K8s infrastructure and GitHub webhooks. |

## 2. Architecture Overview

This design is fully self-hosted and event-driven. GitHub webhooks forward `pull_request` events directly to the self-hosted Letta Server on your Kubernetes cluster. The agent runs inside Kubernetes, clones the repository dynamically (fetching both the PR branch and base branch for diff computation), runs the audit tools locally, posts the advisory comment via GitHub API, and cleans up.

> **Concurrency Warning:** Letta agents are stateful and single-threaded. If multiple PRs are opened simultaneously, they will be processed sequentially. PR #2 and #3 will queue behind PR #1. For high-volume repos, you need either:
> - A **pool of agent IDs** with a load-balancing webhook handler, or
> - A **main agent that dispatches ephemeral worker sub-agents** via Letta's subagent API, or
> - Accept sequential processing and set GitHub Action timeout expectations accordingly.
>
> This plan uses a single agent for simplicity. Upgrade to a pool if PR volume causes timeouts.

```mermaid
flowchart TD
    A[GitHub Pull Request Event] -->|Webhook| B[Kubernetes Ingress / TLS]
    B --> C[Webhook Handler Sidecar]
    C -->|Verify signature| C2[Reject invalid]
    C -->|Valid| D[Letta Server on Kubernetes]
    D --> E[Persistent Volume<br/>MemFS Git Repository]
    D --> DB[(PostgreSQL + pgvector)]

    D --> F[Agent Instance]
    F --> G[Tools: git_clone, check_deps, OSV, Web Search]
    F --> H[Skills: audit, advisory]
    F --> I[Sleep-Time Dream Agent<br/>Background consolidation]

    F -->|Clones PR branch + base| J[Local Pod Workspace<br/>/tmp/workspace]
    F -->|Computes git diff| J
    F -->|Runs check_deps/OSV| J

    F --> K[Posts Advisory Comment<br/>via GitHub API Token]
    K --> L[Updates MemFS on PV]
    F -->|Cleanup| M[Delete /tmp/workspace]
```

### Core Components
1. **GitHub Webhook:** Configured at the repository or organization level to forward `pull_request` events directly to your Kubernetes ingress endpoint.
2. **Webhook Listener / API Gateway:** A lightweight Flask sidecar that receives the webhook, authenticates the GitHub signature, and translates it into a message for the Letta Agent via the REST API.
3. **Letta Server (K8s Deployment):** The agent runtime. Uses the `letta/letta` Docker image (deprecated but functional). Requires PostgreSQL with pgvector as its database backend.
4. **PostgreSQL + pgvector:** Required database for the Letta server. Deployed as a separate StatefulSet or Deployment with its own PVC.
5. **MemFS (Memory File System):** Letta's git-backed memory substrate stored on a K8s Persistent Volume (PVC). Enabled via the `LETTA_MEMFS_SERVICE_URL=local` environment variable. Coexists with memory blocks (always-in-context agent directives).
6. **Memory Blocks:** Editable segments of the agent's system prompt, always loaded into context. Used for `advisory-config` (read-only directives), `team-patterns` (learned patterns), and `dependency-tracking` (flagged deps). These are NOT replaced by MemFS — they serve different purposes.
7. **Sleep-Time Dream Agent:** A built-in background process that wakes up on context compaction triggers to evaluate recent PRs and consolidate memory patterns.
8. **Custom Git Clone Tool:** A tool given to the agent to clone the target PR repository branch AND the base branch locally into the pod's temporary space, enabling `git diff` computation.

## 3. Implementation Plan (1-Day, 5 Stages)

### Stage 0: Agent Initialization (Pre-Deployment)
**Goal:** Create the PR Advisory Agent with its system prompt, skills, and memory blocks before any PR can trigger it.

1. **Port-forward to the Letta server** (once Stage 1 is complete):
   ```bash
   kubectl -n ai-agents port-forward svc/letta-server 8283:8283
   ```

2. **Create the agent** using the Letta SDK or CLI:
   ```bash
   # Using the Letta Python SDK
   pip install letta
   python3 -c "
   from letta import Letta
   client = Letta(base_url='http://localhost:8283/v1')

   # Create the agent with system prompt
   agent = client.agents.create(
       name='PR Advisory Agent',
       memory_blocks=[
           {'label': 'advisory-config', 'value': 'You are strictly advisory. Do not modify code. Do not approve PRs...', 'read_only': True},
           {'label': 'team-patterns', 'value': 'No data yet.', 'read_only': False},
           {'label': 'dependency-tracking', 'value': 'No dependencies flagged yet.', 'read_only': False},
       ]
   )
   print(f'Agent ID: {agent.id}')
   "
   ```

3. **Upload skill files** to the agent's MemFS:
   - Create `.skills/dependency-audit/SKILL.md`
   - Create `.skills/architecture-advisory/SKILL.md`

4. **Store the `agent_id`** in a Kubernetes Secret:
   ```bash
   kubectl -n ai-agents create secret generic letta-agent-config \
     --from-literal=agent-id=<the-returned-agent-id>
   ```

5. **Quality Gate 0:** Verify the agent exists, has the correct system prompt, and has both skills loaded.

### Stage 1: Kubernetes Deployment & Storage (Hours 1-4)
**Goal:** Deploy the Letta Server and PostgreSQL to your cluster with persistent storage.

1. **Create Namespace:**
   ```bash
   kubectl create namespace ai-agents
   ```

2. **Deploy PostgreSQL with pgvector:**
   - **StatefulSet:** Run `ankane/pgvector:v0.5.1` with a PVC for data persistence.
   - **Service:** Expose port 5432 internally.
   - **Secret:** Store PostgreSQL credentials.

3. **Deploy Letta Server:**
   - **Deployment:** Run the `letta/letta:latest` Docker image.
   - **Environment:** Configure `LETTA_PG_URI`, `LETTA_MEMFS_SERVICE_URL=local`, `SECURE=true`, `LETTA_SERVER_PASSWORD`, and LLM API keys.
   - **Service:** Expose port 8283 internally.
   - **PVC:** Provision storage for the MemFS git repository.

4. **Configure Ingress:** Set up an Ingress route with TLS to expose the webhook receiver securely (e.g., `https://letta-webhooks.yourdomain.com/github`).

5. **Quality Gate 1:** Verify both pods are running, PVCs are bound, PostgreSQL is healthy, and you can reach the Letta API endpoint.

### Stage 2: Webhook & Secret Provisioning (Hours 5-8)
**Goal:** Configure GitHub to trigger the self-hosted agent directly.

1. **Create K8s Secrets:** Store your Letta API key, Webhook Verification Secret, GitHub PAT (for posting comments), and LLM API keys in Kubernetes.
   ```bash
   kubectl -n ai-agents create secret generic letta-secrets \
     --from-literal=api-key=<your-letta-api-key> \
     --from-literal=server-password=<your-server-password> \
     --from-literal=github-token=<github-pat-with-pr-comment-scope> \
     --from-literal=webhook-secret=<github-webhook-secret>
   ```

2. **Configure GitHub Webhook:** Register a webhook in your GitHub repository pointing to your Ingress URL. Select event: `Pull requests`. Set the secret to match `webhook-secret`.

3. **Deploy Webhook Handler:** Deploy the Flask webhook handler (see Appendix A) as a sidecar or separate service.

4. **Quality Gate 2:** Trigger a test pull request. Verify that the webhook payload is delivered to Kubernetes, is parsed correctly, and triggers the agent on your server.

### Stage 3: Tooling & Dynamic Workspace Cloning (Hours 9-12)
**Goal:** Implement the repository cloning tool and custom audit scripts.

1. **Build Custom Docker Image:** Create a Dockerfile that extends `letta/letta:latest` and includes:
   ```dockerfile
   FROM letta/letta:latest

   # Install Python and pip
   RUN apt-get update && apt-get install -y python3 python3-pip git

   # Install Python dependencies
   RUN pip3 install requests flask

   # Install osv-scanner (Go binary, NOT pip)
   RUN apt-get install -y golang-go && \
       go install github.com/google/osv-scanner/cmd/osv-scanner@latest && \
       cp /root/go/bin/osv-scanner /usr/local/bin/ && \
       apt-get remove -y golang-go

   # Copy custom tools
   COPY tools/check_deps.py /tools/check_deps.py
   COPY tools/osv_scan.py /tools/osv_scan.py
   COPY tools/git_clone.py /tools/git_clone.py
   COPY tools/augmented_web_search.py /tools/augmented_web_search.py
   ```

2. **Implement Clone Tool:** The `git_clone` tool must fetch BOTH the PR branch and the base branch so the agent can compute `git diff`:
   ```python
   # Clone base branch (e.g., main) AND PR branch
   subprocess.run(["git", "clone", "--bare", repo_url, target_dir], check=True)
   subprocess.run(["git", "--git-dir", target_dir, "fetch", "origin", f"pull/{pr_number}/head:pr-{pr_number}"], check=True)
   # Now the agent can run: git --git-dir=<target_dir> diff main...pr-{pr_number}
   ```

3. **Register Tools:** Register the `git_clone`, `check_deps`, `osv_scan`, and `augmented_web_search` tools with the agent via the Letta API.

4. **Quality Gate 3:** Trigger a PR with an outdated dependency. Verify the agent clones the repo, computes the diff, runs the scripts, posts the comment, and cleans up.

### Stage 4: Memory Consolidation & Polish (Hours 13-16)
**Goal:** Enable sleep-time consolidation and verify persistent memory.

1. **Configure Sleep-Time:** Enable dream agents for self-evaluation. Verify the correct CLI command or API endpoint — check current Letta docs for the exact mechanism. The dream agent should be configured to:
   - Read all files in `history/`
   - Extract recurring patterns into `patterns/`
   - Prune old or irrelevant entries from `history/`
   - Commit changes to the MemFS git repo

2. **Test Memory Persistence:** Open PR #1. Wait for sleep-time consolidation. Open PR #2 referencing the same dependency. Verify the agent recalls the context from PR #1.

3. **Monitor K8s Logs:** Check the pod logs to ensure the sleep-time agent is running without errors and committing to the MemFS git repo on the persistent volume.

4. **Final Polish:** Refine the system prompt to ensure the 5 architectural alternatives are strictly advisory and well-formatted.

### Appendix A: Webhook Receiver Listener (`webhook-handler.py`)
*This runs as a lightweight service or sidecar in the Kubernetes pod to receive GitHub webhooks and trigger the Letta Agent via its REST API.*

```python
from flask import Flask, request, jsonify
import hmac
import hashlib
import os
import requests
import threading

app = Flask(__name__)

LETTA_API_URL = "http://localhost:8283/v1/agents"
AGENT_ID = os.environ.get("LETTA_AGENT_ID")
LETTA_API_KEY = os.environ.get("LETTA_API_KEY")
WEBHOOK_SECRET = os.environ.get("GITHUB_WEBHOOK_SECRET", "").encode()

@app.route("/github", methods=["POST"])
def github_webhook():
    # Verify GitHub Webhook Signature
    signature = request.headers.get("X-Hub-Signature-256")
    if not signature:
        return "Missing signature", 401

    sha_name, signature = signature.split("=")
    mac = hmac.new(WEBHOOK_SECRET, request.data, hashlib.sha256)
    if not hmac.compare_digest(mac.hexdigest(), signature):
        return "Invalid signature", 401

    payload = request.json
    action = payload.get("action")

    # Trigger the agent when a PR is opened or synchronized (updated)
    if "pull_request" in payload and action in ["opened", "synchronize"]:
        pr = payload["pull_request"]
        repo = payload["repository"]

        # Build prompt payload for the agent with full context
        agent_payload = {
            "message": (
                f"Review PR #{pr['number']} in {repo['full_name']}. "
                f"Branch: {pr['head']['ref']}. "
                f"Base: {pr['base']['ref']}. "
                f"Clone URL: {pr['head']['repo']['clone_url']}. "
                f"PR URL: {pr['html_url']}. "
                f"Title: {pr['title']}. "
                f"Body: {pr.get('body', '')}. "
                f"Clone the repo, compute the diff between base and PR branch, "
                f"run dependency analysis, generate 5 architectural alternatives, "
                f"and post an advisory comment. Do not modify code."
            )
        }

        # Trigger the agent turn via Letta API
        # Note: This is synchronous. For concurrency, use a queue or spawn
        # ephemeral worker agents.
        headers = {"Authorization": f"Bearer {LETTA_API_KEY}"}
        try:
            r = requests.post(
                f"{LETTA_API_URL}/{AGENT_ID}/messages",
                json=agent_payload,
                headers=headers,
                timeout=300  # 5-minute timeout for the agent response
            )
            return jsonify({"status": "triggered", "agent_response": r.status_code})
        except requests.exceptions.Timeout:
            return jsonify({"status": "timeout", "message": "Agent processing exceeded 5 minutes"}), 504

    return "Event ignored", 200

if __name__ == "__main__":
    app.run(host="0.0.0.0", port=5000)
```

### Appendix B: `check_deps.py` (Custom Dependency Tool)
```python
#!/usr/bin/env python3
"""Scan repo for dependencies, check latest versions across ecosystems."""
import json
import sys
import requests
from pathlib import Path

def check_npm(repo_path):
    pkg_json = Path(repo_path) / "package.json"
    if not pkg_json.exists(): return []
    deps = json.load(open(pkg_json)).get("dependencies", {})
    results = []
    for name, current_ver in deps.items():
        try:
            r = requests.get(f"https://registry.npmjs.org/{name}/latest", timeout=5)
            latest_ver = r.json().get("version")
            if latest_ver and latest_ver != current_ver:
                results.append({
                    "ecosystem": "npm", "name": name,
                    "current": current_ver, "latest": latest_ver,
                    "update_available": True
                })
        except Exception:
            continue
    return results

def check_pypi(repo_path):
    """Check Python dependencies from requirements.txt or pyproject.toml."""
    results = []
    req_file = Path(repo_path) / "requirements.txt"
    if not req_file.exists(): return results
    for line in req_file.read_text().splitlines():
        line = line.strip()
        if not line or line.startswith("#") or line.startswith("-"):
            continue
        # Parse "package==1.0.0" or "package>=1.0.0"
        parts = line.split("==") if "==" in line else line.split(">=")
        if len(parts) == 2:
            name, current_ver = parts[0].strip(), parts[1].strip()
            try:
                r = requests.get(f"https://pypi.org/pypi/{name}/json", timeout=5)
                latest_ver = r.json().get("info", {}).get("version")
                if latest_ver and latest_ver != current_ver:
                    results.append({
                        "ecosystem": "pypi", "name": name,
                        "current": current_ver, "latest": latest_ver,
                        "update_available": True
                    })
            except Exception:
                continue
    return results

def check_cargo(repo_path):
    """Check Rust dependencies from Cargo.toml."""
    cargo_file = Path(repo_path) / "Cargo.toml"
    if not cargo_file.exists(): return []
    results = []
    try:
        import tomllib
        data = tomllib.loads(cargo_file.read_text())
        deps = data.get("dependencies", {})
        for name, version_spec in deps.items():
            current_ver = str(version_spec).strip('"').lstrip("^").lstrip("~")
            try:
                r = requests.get(f"https://crates.io/api/v1/crates/{name}", timeout=5,
                                 headers={"User-Agent": "PR-Advisory-Agent"})
                latest_ver = r.json().get("crate", {}).get("max_stable_version")
                if latest_ver and latest_ver != current_ver:
                    results.append({
                        "ecosystem": "cargo", "name": name,
                        "current": current_ver, "latest": latest_ver,
                        "update_available": True
                    })
            except Exception:
                continue
    except ImportError:
        pass  # tomllib requires Python 3.11+
    return results

def check_go(repo_path):
    """Check Go dependencies from go.mod."""
    go_mod = Path(repo_path) / "go.mod"
    if not go_mod.exists(): return []
    results = []
    for line in go_mod.read_text().splitlines():
        line = line.strip()
        if line.startswith("require ") and "// indirect" not in line:
            parts = line.replace("require ", "").split()
            if len(parts) >= 2:
                name, current_ver = parts[0], parts[1]
                try:
                    r = requests.get(f"https://proxy.golang.org/{name}/@latest", timeout=5)
                    latest_ver = r.json().get("Version")
                    if latest_ver and latest_ver != current_ver:
                        results.append({
                            "ecosystem": "go", "name": name,
                            "current": current_ver, "latest": latest_ver,
                            "update_available": True
                        })
                except Exception:
                    continue
    return results

if __name__ == "__main__":
    repo = sys.argv[1]
    all_deps = check_npm(repo) + check_pypi(repo) + check_cargo(repo) + check_go(repo)
    print(json.dumps(all_deps, indent=2))
```
