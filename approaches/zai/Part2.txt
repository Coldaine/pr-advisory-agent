## 1. Agent Persona & Skill Definitions

The agent's behavior is governed by a core system prompt and two specialized skills loaded into its context.

### Core System Prompt
*Instructions: This is injected at the agent creation level via the Letta CLI or SDK. See Stage 0 in Part 1 for the creation script.*

```text
You are an automated PR Advisory Agent. Your role is strictly advisory.
You DO NOT modify source code, push commits, or approve PRs.

For every PR triggered:
1. Parse the incoming message to extract the clone URL, branch, base branch, and PR number.
2. Execute the `git_clone` tool to clone the PR branch AND base branch to `/tmp/workspace` inside your container.
3. Compute the git diff between the base branch and the PR branch to understand what changed.
4. Load the `dependency-audit` skill and execute dependency analysis against `/tmp/workspace`.
5. Load the `architecture-advisory` skill and execute architectural analysis against `/tmp/workspace` and the diff.
6. Generate a single, consolidated markdown comment containing:
   - Dependency Version Analysis (Current vs. Latest)
   - Security Vulnerabilities (if any)
   - 5 Architectural Alternatives with Pros/Cons
7. Post the comment to the pull request using your GitHub API token.
8. Execute the workspace cleanup tool to delete `/tmp/workspace`.
9. Save a summary of this PR's analysis to your MemFS (memory).
```

### Skill 1: `dependency-audit.md`
*Location: Stored in `.skills/dependency-audit/SKILL.md` (auto-discovered by Letta Code).*

```markdown
# Skill: Dependency Audit

## When to use
Triggered for every PR analysis to check dependency health.

## Instructions
1. Run the dependency script: `python3 /tools/check_deps.py /tmp/workspace`
2. Run the security scan: `osv-scanner --lockfile /tmp/workspace/package-lock.json --format json` (or appropriate lockfile for the ecosystem)
3. Optionally cross-reference outdated dependencies with the `augmented_web_search` tool.
   - Search for: "<package_name> changelog v<latest_version>"
   - Use web search to scrape changelog URLs for specific breaking changes.
4. Format findings as a markdown table with columns:
   - Package | Current | Latest | Security Risk | Breaking Changes
5. Do not suggest upgrading if the current version is a security patch and the latest is a major version jump without clear migration paths.
```

*Location: Stored in `.skills/architecture-advisory/SKILL.md` (auto-discovered by Letta Code).*

```markdown
# Skill: Architectural Advisory

## When to use
Triggered for every PR to reason over system design.

## Instructions
1. Inspect the cloned source code under `/tmp/workspace`.
2. Review the git diff to understand what the PR actually changes.
3. Check MemFS `patterns/` directory for known architectural patterns of this repo.

   - **Cons**: 3 bullet points.

6. Be realistic. If the PR is a minor config change, suggest architectural improvements to the surrounding system context rather than the config itself.
```

## 2. Custom Tool Implementations

These scripts must be packaged into the custom Docker image built on top of `letta/letta:latest` (not `letta/letta-server` — that image does not exist).

### `git_clone.py`
*Clones the target PR branch AND base branch locally into the pod container workspace. Fetches both so the agent can compute `git diff`.*

```python
#!/usr/bin/env python3
"""Clone a PR branch and base branch for diff computation."""
import subprocess
import sys
import os
import shutil
import json

def clone(repo_url: str, pr_branch: str, base_branch: str, pr_number: str, target_dir: str = "/tmp/workspace"):
    """Clones the repo and fetches both the PR branch and base branch."""
    if os.path.exists(target_dir):
        shutil.rmtree(target_dir)

    # Inject GitHub Token for authentication
    token = os.environ.get("GITHUB_TOKEN")
    if token and "github.com" in repo_url:
        repo_url = repo_url.replace("https://", f"https://x-access-token:{token}@")

    try:
        # Clone bare to have full ref space for diffing
        subprocess.run(
            ["git", "clone", "--bare", repo_url, target_dir],
            check=True, capture_output=True, text=True
        )

        # Fetch the PR ref
        subprocess.run(
            ["git", "--git-dir", target_dir, "fetch", "origin",
             f"pull/{pr_number}/head:pr-{pr_number}"],
            check=True, capture_output=True, text=True
        )

        # Ensure base branch is available
        subprocess.run(
            ["git", "--git-dir", target_dir, "fetch", "origin", base_branch],
            check=True, capture_output=True, text=True
        )

        return {
            "status": "success",
            "message": f"Cloned repo. PR branch: pr-{pr_number}, base: {base_branch}",
            "diff_command": f"git --git-dir={target_dir} diff {base_branch}...pr-{pr_number}"
        }
    except subprocess.CalledProcessError as e:
        return {"status": "error", "message": e.stderr}

if __name__ == "__main__":
    url = sys.argv[1]        # clone URL
    pr_branch = sys.argv[2]  # PR head branch
    base_branch = sys.argv[3] if len(sys.argv) > 3 else "main"
    pr_num = sys.argv[4] if len(sys.argv) > 4 else "1"
    dest = sys.argv[5] if len(sys.argv) > 5 else "/tmp/workspace"
    print(json.dumps(clone(url, pr_branch, base_branch, pr_num, dest), indent=2))
```

### `augmented_web_search.py`
*Aggregates Exa and Tavily for web intelligence. Firecrawl is optional — direct package registry queries are preferred for version checking.*

```python
#!/usr/bin/env python3
import os
import requests
import json
import sys

def search(query: str, search_type: str = "semantic"):
    """
    Routes search to appropriate provider.
    search_type: 'semantic' (Exa), 'general' (Tavily)
    """
    if search_type == "semantic":
        r = requests.post(
            "https://api.exa.ai/search",
            headers={"x-api-key": os.environ["EXA_API_KEY"]},
            json={"query": query, "num_results": 3, "contents": {"text": True}},
            timeout=10
        )
        return r.json()

    elif search_type == "general":
        r = requests.post(
            "https://api.tavily.com/search",
            headers={"Authorization": f"Bearer {os.environ['TAVILY_API_KEY']}"},
            json={"query": query, "max_results": 3},
            timeout=10
        )
        return r.json()

if __name__ == "__main__":
    query = sys.argv[1]
    search_type = sys.argv[2] if len(sys.argv) > 2 else "semantic"
    print(json.dumps(search(query, search_type), indent=2))
```

### `osv_scan.py`
*Wrapper for Google's OSV-Scanner to check for vulnerabilities. OSV-Scanner is a Go binary installed in the Docker image, NOT a Python package.*

```python
#!/usr/bin/env python3
"""Runs OSV-Scanner (Go binary) and returns JSON output."""
import subprocess
import json
import sys
import os
from pathlib import Path

def scan(repo_path: str):
    """Runs OSV-Scanner against all lockfiles found in the repo."""
    results = []

    # Find all lockfiles OSV-Scanner can handle
    lockfiles = []
    for pattern in ["package-lock.json", "yarn.lock", "pnpm-lock.yaml",
                     "poetry.lock", "requirements.txt", "Cargo.lock",
                     "go.sum", "Gemfile.lock"]:
        found = list(Path(repo_path).rglob(pattern))
        lockfiles.extend(str(f) for f in found)

    if not lockfiles:
        return {"status": "no_lockfiles", "message": "No supported lockfiles found"}

    for lockfile in lockfiles:
        try:
            result = subprocess.run(
                ["osv-scanner", "--lockfile", lockfile, "--format", "json"],
                capture_output=True, text=True, timeout=30
            )
            if result.stdout:
                parsed = json.loads(result.stdout)
                parsed["lockfile"] = lockfile
                results.append(parsed)
        except subprocess.TimeoutExpired:
            results.append({"error": f"Timeout scanning {lockfile}"})
        except json.JSONDecodeError:
            results.append({"error": f"Invalid JSON from scanning {lockfile}"})
        except Exception as e:
            results.append({"error": str(e)})

    return {"results": results}

if __name__ == "__main__":
    repo = sys.argv[1]
    print(json.dumps(scan(repo), indent=2))
```

## 4. Memory Architecture: MemFS + Memory Blocks

MemFS and memory blocks are complementary mechanisms that coexist in the Letta architecture. MemFS does NOT replace memory blocks.

### Memory Blocks (Always in Context)
These are editable segments of the agent's system prompt, always loaded, always visible:
- `advisory-config` (read-only): Directives for how to review PRs, format requirements
- `team-patterns` (read-write): What the team acts on vs ignores, updated after each review
- `dependency-tracking` (read-write): Currently flagged dependencies and their status

### MemFS Structure (Git-Tracked, On-Demand)
MemFS is enabled via the `LETTA_MEMFS_SERVICE_URL=local` environment variable (NOT a CLI command).

```text
/root/.letta/
├── system/
│   ├── config.md              # Agent persona and global config
│   └── skills/                # Wait — skills go in .skills/ at repo root, not here
├── history/                   # Empty initially
│   └── pr-42.md               # Detailed log of PR 42 analysis (after first PR)
└── patterns/                  # Empty initially
    ├── auth-strategies.md     # Updated: noticed basic auth used, noted JWT alternative
```

## 5. Operational Checklist & Troubleshooting

### Pre-Flight Checklist
- [ ] K8s namespace `ai-agents` exists
- [ ] PostgreSQL StatefulSet is running and healthy (`pg_isready`)
- [ ] Letta server pod is running with all env vars configured
- [ ] `LETTA_MEMFS_SERVICE_URL=local` is set (MemFS enabled)
- [ ] `SECURE=true` and `LETTA_SERVER_PASSWORD` are set
- [ ] K8s PVCs are bound (both MemFS and PostgreSQL)
- [ ] Letta server API is reachable via Ingress/port-forward
- [ ] Agent is created with correct system prompt and memory blocks (Stage 0)
- [ ] Skills are in `.skills/dependency-audit/SKILL.md` and `.skills/architecture-advisory/SKILL.md`
- [ ] `osv-scanner` is installed as Go binary in the custom Docker image
- [ ] Custom tools (`check_deps.py`, `osv_scan.py`, `git_clone.py`) are in `/tools/` in the container
- [ ] GitHub webhook is configured pointing to Ingress URL
- [ ] GitHub webhook secret matches K8s secret
- [ ] GitHub token (for posting comments) is in K8s secret and scoped to `pull-requests: write` only
- [ ] Ingress TLS is configured

### Concurrency Warning
If multiple PRs are opened simultaneously, they will be processed sequentially by the single agent. For high-volume repos:
- Deploy multiple agent instances with different `agent_id`s
- Use a queue (e.g., Redis) in the webhook handler to serialize requests
- Or use Letta's subagent API to spawn ephemeral worker agents

### Monitoring Context Rot
If the agent starts behaving erratically or ignoring instructions:
1. Check the context window size in the Letta ADE (Agent Development Environment)
2. If context is >90% full, manually trigger compaction: `/compact`
3. Check the dream agent logs to ensure it is successfully pruning `history/`
4. Inspect memory blocks via the Letta API to verify `advisory-config` is intact

### GitHub API Rate Limits
The GitHub token used for posting comments has a limit of 5,000 requests/hour (for PATs) or higher for GitHub App installation tokens. Posting one comment per PR uses one API call, so this is unlikely to be a bottleneck. Monitor for 403 errors in the webhook handler logs.

### Docker Image Deprecation
The `letta/letta` Docker image is no longer actively maintained. Risks:
- No security patches for the container itself
- No bug fixes or feature updates
- May break with future Kubernetes or PostgreSQL versions

Mitigations:
- Pin to a specific version tag instead of `:latest`
- Monitor the `letta-ai/letta` repo for any final updates
- Consider migrating to Letta Code local mode on a persistent machine
